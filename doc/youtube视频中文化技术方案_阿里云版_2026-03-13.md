## 🎯 详细步骤说明

### Step 1: 视频下载与音频提取

**功能**: 下载 YouTube 视频并提取音频轨道

**技术选型**:
- **视频下载**: `youtube-dlp` (通过 Node.js `child_process` 调用)
- **音频提取**: `ffmpeg`
- **Node 库**:
  - `fluent-ffmpeg`
  - `@ffmpeg-installer/ffmpeg`
  - `child_process` (内置)

**流程**:
```
YouTube URL
    → yt-dlp 下载视频 (mp4)
    → ffmpeg 提取音频 (mp3/wav)
    → 音频文件
```

**推荐配置**:
- 音频格式: `wav` (无损，适合语音识别)
- 采样率: `16kHz` (通义听悟推荐)
- 音频时长: 单次识别建议 ≤ 5 小时

---

### Step 2: 语音识别 (英文转文字) 🆕

**功能**: 将英文音频转换为带时间戳的文本

**推荐模型**: **`paraformer-v1`**

**模型特性**:
- ✅ 通义实验室自研语音识别模型
- ✅ 支持中英文混合识别
- ✅ 高准确率 (98%+)
- ✅ 支持时间戳输出
- ✅ 支持实时识别 (paraformer-realtime-v1)

**备选模型**:
- `paraformer-realtime-v1` - 实时识别版本
- `sense-voice` - 新一代语音识别 (如果可用)

**API 调用方式**:
```javascript

  Authorization: `Bearer ${DASHSCOPE_API_KEY}`,

}
Body: {
  model: "paraformer-v1",
  input: {
    file_url: "https://your-audio-file.wav"
  }

```

**SDK 调用**:

const result = await client.audio.transcription({

**输出格式**:
- **文本**: 识别后的英文文本
- **时间戳**: 每句话的开始和结束时间
- **置信度**: 每段识别的置信度分数

**注意事项**:
- 音频文件大小限制: 单文件 ≤ 512MB
- 支持格式: `wav`, `mp3`, `aac`, `flac`
- 建议预处理: 降噪、统一音量

### Step 3: 翻译 (英文文本转中文文本) 🆕

**功能**: 将英文文本翻译成中文，保持语义和时间戳对齐

**推荐模型**: **`qwen-plus`**

**模型选择对比**:

| 模型 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **qwen-max** | 翻译质量最高，语义理解最强 | 价格较高，响应稍慢 | 专业翻译、字幕制作 |
| **qwen-plus** ⭐ | 性价比最佳，质量优秀 | - | 日常翻译、推荐使用 |
| **qwen-turbo** | 速度快，成本低 | 翻译质量稍逊 | 快速预览、草稿翻译 |

**Prompt 模板**:
```text
你是一个专业的视频字幕翻译官。请将以下英文文本翻译成中文，要求：

1. 保持口语化和自然流畅
2. 保留原始句子的情感和语气
3. 控制句子长度，避免过长（适合视频字幕显示）
4. 专业术语保持准确
5. 不要添加任何额外解释

英文原文：
{english_text}

中文翻译：
```

**API 调用方式**:
```javascript
POST https://dashscope.aliyuncs.com/api/v1/services/aigc/text-generation/generation
Headers: {
  Authorization: `Bearer ${DASHSCOPE_API_KEY}`,
  Content-Type: 'application/json'
}
Body: {
  model: "qwen-plus",
  input: {
    prompt: "翻译 prompt..."
  },
  parameters: {
    max_tokens: 2000,
    temperature: 0.3
  }
}
```

**SDK 调用**:
```javascript
const result = await client.generation.call({
  model: 'qwen-plus',
  prompt: translationPrompt,
  parameters: {
    max_tokens: 2000,
    temperature: 0.3
  }
});
```

**时间戳处理**:
- 翻译时**不要改变句子结构**
- 保持每句话独立翻译
- 中文句子长度可能与英文不同，需要在 Step 5 进行音频拉伸调整

**优化建议**:
- 分段翻译: 每 10-20 句为一组，避免 token 超限
- 上下文保持: 在 prompt 中加入前几句作为上下文
- 术语一致性: 对于专业术语，可以在 prompt 中提供术语表

### Step 5: 音频处理与视频合成

**功能**: 将中文语音替换原视频音轨，添加中文字幕

**技术选型**:
- **工具**: `ffmpeg`
- **Node 库**: `fluent-ffmpeg`

**处理步骤**:

#### 5.1 音频拼接
```bash
# 生成音频片段列表文件
cat > audio_list.txt << EOF
file 'segment_001.mp3'
file 'segment_002.mp3'
file 'segment_003.mp3'
EOF

# 拼接音频
ffmpeg -f concat -safe 0 -i audio_list.txt -c copy chinese_audio.mp3
```

#### 5.2 音频拉伸 (同步时间轴)
```bash
# 如果中文语音时长与原视频不同，需要拉伸/压缩
# 计算速度系数: 原时长 / 中文时长
ffmpeg -i chinese_audio.mp3 -filter:a "atempo=1.05" adjusted_audio.mp3
```

#### 5.3 添加字幕
```bash
# 烧录字幕到视频 (硬字幕)
ffmpeg -i original.mp4 -vf "subtitles=chinese_subtitle.srt:force_style='Fontsize=24,PrimaryColour=&H00FFFFFF'" -c:a copy output_with_subtitles.mp4

# 或保留软字幕
ffmpeg -i original.mp4 -i chinese_subtitle.srt -c copy -c:s mov_text output_soft_subtitle.mp4
```

#### 5.4 替换音轨 + 最终合成
```bash
ffmpeg -i original_video.mp4 -i adjusted_audio.mp3 \
  -map 0:v -map 1:a \
  -c:v copy -c:a aac \
  -shortest \
  output_chinese_video.mp4
```

#### 5.5 完整合成 (一步到位)
```bash
ffmpeg -i original_video.mp4 \
  -i adjusted_audio.mp3 \
  -vf "subtitles=chinese_subtitle.srt" \
  -map 0:v:0 -map 1:a:0 \
  -c:v libx264 -preset medium -crf 23 \
  -c:a aac -b:a 192k \
  -shortest \
  final_output.mp4
```

**Node.js 代码示例**:
```javascript
const ffmpeg = require('fluent-ffmpeg');

ffmpeg()
  .input('original.mp4')
  .input('chinese_audio.mp3')
  .complexFilter([
    {
      filter: 'subtitles',
      options: { filename: 'chinese_subtitle.srt' }
    }
  ])
  .outputOptions([
    '-map 0:v:0',
    '-map 1:a:0',
    '-c:v libx264',
    '-preset medium',
    '-crf 23',
    '-c:a aac',
    '-b:a 192k',
    '-shortest'
  ])
  .save('final_output.mp4')
  .on('end', () => {
    console.log('视频合成完成!');
  });
```

## ⚠️ 难点与解决方案

### 1. 时间同步问题 ⭐ 最重要

**问题**: 中文语音长度通常与英文不同，导致音画不同步

**解决方案**:

#### 方案 A: 音频拉伸 (推荐)
```bash
# 计算拉伸系数
原视频时长 = 600 秒
中文语音时长 = 630 秒
拉伸系数 = 600 / 630 = 0.952

# 应用拉伸
ffmpeg -i chinese_audio.mp3 -filter:a "atempo=0.952" synchronized_audio.mp3
```

#### 方案 B: 重新计算时间轴
```javascript
// 根据中文语音实际时长，重新分配时间戳
const adjustTimestamps = (sentences, totalDuration) => {
  let currentTime = 0;

  return sentences.map(sentence => {
    const duration = sentence.chineseDuration;
    const adjustedSentence = {
      ...sentence,
      start: currentTime,
      end: currentTime + duration
    };
    currentTime += duration;
    return adjustedSentence;
  });
};
```

#### 方案 C: 分段拉伸
```javascript
// 对每句话单独拉伸，保持相对节奏
const stretchPerSentence = (englishSentences, chineseSentences) => {
  return chineseSentences.map((chinese, index) => {
    const english = englishSentences[index];
    const stretchRatio = english.duration / chinese.duration;
    return { ...chinese, stretchRatio };
  });
};
```

### 2. 成本控制

| 服务 | 定价 | 10分钟视频估算 |

### 4. 音质一致性

**问题**: 合成语音与原视频背景音不协调

**解决方案**:

#### 4.1 保留背景音乐
```bash
# 分离人声和背景音乐
ffmpeg -i original.mp4 -filter_complex "vocal_remover" background_music.mp3

# 混合合成语音和背景音乐
ffmpeg -i synthesized_speech.mp3 -i background_music.mp3 \
  -filter_complex "amix=inputs=2:duration=first" \
  final_audio.mp3
```

#### 4.2 音量平衡
```javascript
// 调整合成语音的音量
const adjustVolume = (audioPath: string, targetDb: number) => {
  return ffmpeg(audioPath)
    .audioFilters(`volume=${targetDb}dB`)
    .save(`${audioPath}.adjusted.mp3`);
};
```

### 5. 字幕格式处理

**问题**: 时间戳格式转换和对齐

**SRT 格式生成**:
```javascript
function generateSRT(sentences: any[]) {
  return sentences.map((sentence, index) => {
    const start = formatTime(sentence.start);
    const end = formatTime(sentence.end);
    return `${index + 1}\n${start} --> ${end}\n${sentence.text}\n`;
  }).join('\n');

  function formatTime(seconds: number): string {
    const hrs = Math.floor(seconds / 3600);
    const mins = Math.floor((seconds % 3600) / 60);
    const secs = Math.floor(seconds % 60);
    const ms = Math.floor((seconds % 1) * 1000);
    return `${pad(hrs)}:${pad(mins)}:${pad(secs)},${pad(ms, 3)}`;
  }
}
```

    message: string,

// API 调用重试机制
export async function callWithRetry<T>(
  fn: () => Promise<T>,
  maxRetries = 3
): Promise<T> {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn();
    } catch (error) {
      if (i === maxRetries - 1) throw error;
      await sleep(1000 * Math.pow(2, i)); // 指数退避
    }
  }
  throw new Error('Max retries exceeded');
}
